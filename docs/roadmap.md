总计约 34-50 轮，外加真实玩家测试周期。模型名称以当前 Codex 可选项为准；若 UI 中版本更新，遵循“最新高推理模型做架构/消歧，最新 Codex 编码模型做实现/验证”的角色分配。

## 执行顺序与并行纪律

- Phase 1 前先冻结 `GameState / Command / DomainEvent / Ruleset` 的最小契约。
- 同一时刻只允许一个 worker 改公共领域契约；其余 worker 基于测试夹具并行。
- 每个 Phase 按“目标与验收 -> 实现 -> 自测 -> 架构自检 -> 人工验收 -> ADR/roadmap 更新”闭环。
- 任一门禁失败不得进入下一 Phase；真实玩法判断必须由玩家测试拍板，模型模拟只用于发现边界。
