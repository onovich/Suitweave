# 织花牌 / Suitweave UI/UX 重构设计契约

日期：2026-07-13  
状态：DesignByGPT studio-loop 前向测试的执行基线

## 1. 设计 thesis

**“三盘织局 / Loom Table”——玩家不是在操作三个并列仪表盘，而是在一张深色织机桌上，把同一手牌的纹样与风险织进三幅彼此竞争的牌阵。**

这套方向不可直接移植到普通 SaaS：棋盘是绷在织框中的织面；合法连通组由“丝线路径”而非普通高亮表达；手牌像象牙梭片；皇冠与灵感像金属绣章；“选牌 → 试织 → 落针”构成唯一行动语法。

方向名：**三盘织局 / Loom Table**。

反目标：

- 不做三张同权重 dashboard 卡片；主操作盘必须明显占据注意力。
- 不做霓虹赛博、紫色渐变、玻璃拟态或通用 roguelike 木板皮肤。
- 不靠大段说明文、内部枚举值或颜色单独解释风险。
- 不用 emoji 充当正式图标；花色、皇冠、灵感使用可控的文字字形或本地 SVG。
- 不以拖拽作为唯一入口；点击、触摸和键盘共享“选牌 → 试织 → 确认”。
- 不修改领域规则、seed、计分、奖励语义或 preview/execution 同源路径。

## 2. 视觉 token

### 2.1 颜色

| 语义 | Token | 值 | 用途 |
| --- | --- | --- | --- |
| 桌面底色 | `--loom-bg` | `#0B1210` | 页面与沉浸背景 |
| 织机表面 | `--loom-surface` | `#14201C` | 主棋盘与固定控制区 |
| 次级织面 | `--loom-surface-muted` | `#1B2924` | 非活动棋盘、备区 |
| 象牙牌面 | `--tile-ivory` | `#F3E6CA` | 手牌与关键数字 |
| 墨黑牌字 | `--tile-ink` | `#17201D` | 浅色牌面文字 |
| 金线 | `--thread-gold` | `#D6AD57` | 皇冠、确认、有效连接 |
| 警示铜 | `--warning-copper` | `#D57952` | 过载/爆点/不可逆风险 |
| 成功青 | `--success-jade` | `#69A68A` | 可提交、成功反馈 |
| 边框线 | `--loom-border` | `#465950` | 结构边界 |
| 主文字 | `--text-primary` | `#F5EBD8` | 标题与正文 |
| 次文字 | `--text-muted` | `#AEBDB4` | 辅助信息 |

四种墨色继续使用红/蓝/绿/紫，但每种必须同时绑定独有纹理；任何风险、选择、锁定状态还需文字、图形或位置线索。

### 2.2 字体

- 展示/盘名：`Iowan Old Style`, `Baskerville`, `Noto Serif SC`, `Songti SC`, serif；中等字重，克制使用。
- 正文/控件：`Inter`, `Noto Sans SC`, `Microsoft YaHei`, sans-serif。
- 数字/计分：`ui-monospace`, `SFMono-Regular`, `Cascadia Mono`, monospace；使用等宽数字。

不引入网络字体或第三方授权资产。

### 2.3 几何与材质

- 间距：4/8/12/16/24/32px；不增加无来源的离散值。
- 圆角：棋盘织框 14px；控制面 10px；梭片手牌 8px；小状态 4px。
- 边线：常规 1px；活动织框 2px 金线；焦点 3px 高对比外描边。
- 阴影：只用桌面接触阴影和手牌抬起阴影两级，禁止每块内容各自悬浮。
- 动效：选择抬起 120ms；试织丝线 160ms；确认落针 180ms；面板切换 180–220ms。动画不阻塞输入，reduced-motion 下改为即时状态切换。

## 3. 布局与交互契约

宽屏（≥1180）：

```text
┌ 品牌/本局目标 ───────────────────── 回合·行动·声音 ┐
├──────────── 主织框 56–60% ───────────┬─ 两盘摘要 ─┤
│ 盘名/分数/风险      7×7 织面          │ [21点缩略] │
│ 同色连通丝线、目标格、皇冠路径         │ [三消缩略] │
├───────────────────────────────────────┴────────────┤
│ 手牌梭片（当前选择抬起） │ 试织结果 │ 主确认动作   │
└────────────────────────────────────────────────────┘
```

1024/768：主盘优先；另外两盘显示紧凑摘要与切换按钮，三盘分数、锁定和风险数量持续可见。

390/320：单盘分页；顶部三盘状态轨道；7×7 网格在 320px 内完整可见且页面 `scrollWidth <= clientWidth`；底部行动坞整合手牌、牌面选择、预览和确认，禁止多个固定浮层互相遮挡。

### 3.1 唯一行动语法

1. **选牌**：梭片抬起，出现“已选 6♦ · 绿色”；有效目标获得轮廓，禁止目标保持可理解的禁用形态。
2. **试织**：点击/Enter 落点只生成领域 preview；格内出现幽灵牌，受影响连接组显示丝线路径。
3. **读结果**：主反馈使用玩家语言，例如“德扑盘 +18”“形成 4 张同色组”；风险使用“孤张”“6/5 过载”“22/21 爆点”，并提供图形符号。
4. **确认**：同一行动坞内显示“确认落牌”；Esc/“取消试织”回到选牌态；确认仍调用现有 execution path。

键盘焦点只能移动焦点，**不得因 `onFocus` 触发 preview 或改变 GameSession**。方向键网格导航为建议项；最低门槛是自然 Tab、Enter/Space 可操作与 Esc 取消预览。

### 3.2 格子四层

1. 墨色底与纹理；2. 连通丝线/组边界；3. 数字与花色；4. 皇冠、灵感、锁定、幽灵/风险状态。层级使用伪元素或专用呈现组件，不在 UI 内复制领域判断。

### 3.3 状态词汇

| 状态 | 表达 |
| --- | --- |
| loading | 保留织框尺寸的低对比骨架；不使用全屏转圈 |
| empty | 空织格 + “可落牌”/目标提示 |
| success | 玉色边线 + 对勾/“可提交”文案 |
| warning | 铜色断线/三角形 + 明确风险名称与数值 |
| error | 原位错误说明 + 恢复动作；不只 toast |
| selected | 金线外框、梭片抬起、`aria-pressed=true` |
| disabled | 降低材质对比、保留可读标签与原因 |
| locked | 织框扣锁图形 + “已锁定” |

## 4. 基线证据与优先修复

固定 seed `20260711`，初始第 1 回合：

- `docs/visual-evidence/designbygpt-baseline/baseline-{1440,1024,768,390,320}.png`
- `docs/visual-evidence/designbygpt-baseline/baseline-390-preview.png`
- 320px：`scrollWidth 354 > clientWidth 305`，存在水平溢出。
- “Mute sound” 高 26px，未达到主移动控件 44px 门槛。
- 宽屏三盘同权，教程横条与声音原生控件抢占主任务之前的空间。
- 预览风险泄漏 `singleton · breaks-valid-group` 内部词汇。
- 焦点事件调用 `targetCell`，键盘浏览有改变游戏预览状态的风险。
- 初始态与一次试织过程控制台无 warning/error。

## 5. 可量化验收门

- 同 seed、同回合、同选牌/试织状态重新截取 1440/1024/768/390/320；每档都有 before/after。
- 320 和 390：`scrollWidth <= clientWidth`；关键流程无需横向页面滚动；网格不裁切。
- 1440：活动盘面积或视觉权重明显高于任一摘要盘；首屏无需阅读教程即可找到“手牌 → 盘面 → 确认”。
- 所有主要按钮/手牌/盘切换在窄屏至少 44×44 CSS px。
- Tab 只移动焦点；Enter/Space 可选牌与确认；Esc 可取消 preview；焦点始终可见。
- 正常与 `prefers-reduced-motion: reduce` 均可完成选牌 → 试织 → 取消/确认；关键反馈不依赖动画。
- 玩家可在不理解英文内部枚举的情况下识别成功、风险、不可用、已选与锁定。
- 实测主路径无未捕获 console error；现有固定 seed E2E、`pnpm test:e2e`、`pnpm validate`、domain platform scan、`git diff --check` 全通过。
- `ui -> application -> domain` 不变；不修改领域规则或复制 preview 计算；若公共契约/架构变化则停止并补 ADR。
- Rendered Frontend Review Rubric 每个关键维度 ≥2，平均分 ≥2.2；Product identity、Hierarchy、Interaction clarity、Distinctiveness 目标 ≥3。
