# 织花牌 / Suitweave：产品与 UI 设计

> GDD 中的《三盘连彩》是旧名。所有新增代码、界面与传播材料统一使用“织花牌 / Suitweave”。原 PDF 仅作为历史规则来源保留。

## 体验判断

核心乐趣不是单盘最优解，而是同一张牌在三盘间的机会成本：抽 6 用 3，把数字织入 2-5 个数字的同色连通组，在 24 回合中不断权衡完成、提分与奖励路线。UI 的最高优先级因此是“行动后果预览”，其次才是动画。

## 视觉方向

- 主题：深色织机桌面、象牙牌面、四种高区分墨色、克制的金色皇冠。
- 语义层分离：墨色用底纹/粗边；花色用角标；皇冠用金色描边与图标；危险用形状加文字，绝不只依赖颜色。
- 动效：落牌 120-180ms、连通域沿路径扩散、有效组轻微收束；警告不抖屏。
- 字体：标题采用具有纸牌感的衬线体，规则与数值采用高可读无衬线体；数字必须等宽。

## 响应式信息架构

- 宽屏（>=1180px）：三盘横排，底部固定手牌与备牌区，右上常驻回合/行动数。
- 中屏：主盘放大、另外两盘缩略并可一键切换；状态摘要始终同时可见。
- 窄屏（<700px）：单盘分页而非三盘长滚动；底部手牌横滑；顶栏保留三盘完成度与警告数。
- 键鼠、触摸、手柄共享“选牌 -> 预览 -> 确认”状态机；拖拽只是快捷交互，不是唯一入口。

## 关键界面

1. 主局面：三盘、回合资源、手牌、功能牌、灵感条。
2. 行动预览：高亮受影响连通域，显示前后组规模、牌型/点数、分差、过载/爆点/落单。
3. 奖励选择：完成/皇冠 6 选 3；灵感 3 选 1；明确行动成本与目标类型。
4. 提交确认：逐组结算、总分、不可逆锁定说明。
5. 结算与复盘：完成盘数、评级、关键转折、种子与可复现统计。

## 可用性门槛

- 所有信息可由键盘访问，焦点清晰；触控目标至少 44px。
- 色盲模式为四种墨色增加纹理；警告同时使用图标、文案、边框。
- 预览计算必须与实际规则使用同一个领域函数，禁止 UI 重写一份计分逻辑。
